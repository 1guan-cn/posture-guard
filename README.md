# posture-guard

> 本地摄像头颈椎前倾与久坐监测。每 3 秒一帧，纯几何判定，全程离线，不上传任何画面。

久坐和低头是程序员的两大慢性损伤源。市面上的提醒类工具要么是计时器（不知道你是不是真的离座了），要么是依赖云端 AI（隐私堪忧）。这个项目用一个 6.5 MB 的本地姿态模型 + 一些朴素几何判定，在 CPU 上每秒 < 2% 占用，做到三件事：

- 久坐 40 分钟提醒（**真离座**才重置计时，短暂走神不算）
- 颈椎前倾 60 秒提醒
- 严重低头 30 秒提醒

模型只在本地推理，画面不落盘、不联网。

---

## 工作原理

每 3 秒抓一帧 → `yolov8n-pose` 输出 17 个人体关键点 → 几个简单几何指标 → 滑动窗口去抖 → 触发提醒。

### 颈椎前倾如何判定

定义 **头肩高度比**：

```
ratio = (肩中点 y - 头中点 y) / 肩宽
```

- 头中点 = 鼻 + 双眼 + 双耳 中所有可见点 y 坐标的平均（最多 5 个点取均值，单点丢失/抖动影响极小）
- 端正坐姿 ≈ **0.7+**（头明显高于肩）
- 轻度低头 ≈ **0.5**
- 严重低头 ≈ **0.3**
- 趴桌 ≈ **负值**

⚠️ 早期版本用 `atan2` 算颈椎与垂直方向的夹角，但 atan2 对水平噪声极为敏感（左右耳交替遮挡、关键点亚像素抖动会被放大成 ±20° 跳动）。改用纯垂直比值后，对水平噪声完全免疫。

### 时序平滑

原始 ratio 仍有少量抖动，再叠一层 EMA：

```
ema = 0.4 * raw + 0.6 * prev_ema
```

判定基于 ema 而不是 raw。

### 滑动窗口防误报

- **轻度前倾**：ema < 0.5，最近 20 帧（60 秒）有 70% 命中才触发
- **严重低头**：ema < 0.3，最近 10 帧（30 秒）有 70% 命中才触发

### 久坐计时

只在"检测到人"时累加：

- 离座 < 10 分钟：回来后**续计**（短暂喝水、找东西不算有效活动）
- 离座 ≥ 10 分钟：回来后**重置为 0**（算作真正起来活动了）
- 任何提醒同类 5 分钟冷却，不会刷屏

## 安装

需要 macOS（系统通知依赖 `osascript` + `afplay`）和 Python 3.11+。

```bash
git clone https://github.com/<your-user>/posture-guard.git
cd posture-guard
uv sync   # 或 pip install -r <(uv pip compile pyproject.toml)
```

依赖：`ultralytics`、`opencv-python`、`numpy`。模型文件 `yolov8n-pose.pt`（6.5 MB）首次启动自动下载。

## 运行

```bash
uv run main.py
```

首次启动 macOS 会请求摄像头权限；授权后控制台每 3 秒输出一行：

```
[09:12:08] 在座 459s 头肩比 0.65/平滑 0.62
[09:12:11] 在座 462s 头肩比 0.45/平滑 0.55 前倾
[09:12:14] 在座 465s 头肩比 0.18/平滑 0.31 严重低头!
```

`Ctrl+C` 退出。

## 提醒方式

弹出**模态对话框**（必须点"知道了"）+ Hero 系统音。10 分钟无操作自动关闭兜底，避免挂死。

如不想要模态弹窗，把 `main.py` 里的 `notify()` 改回 `display notification` 即可（变成系统通知中心横幅）。

## 提醒规则一览

三类提醒，全部同类 5 分钟冷却。

| 类型 | 触发条件 | 文案 |
|---|---|---|
| **久坐** | 连续在座 40 分钟 | 该起来活动了 — 已经连续坐 40 分钟，起来走走吧 |
| **轻度前倾** | 最近 60 秒（20 帧）内 ≥70% 帧 `ema_ratio < 0.5` | 调整坐姿 — 颈部前倾过久，挺直背 |
| **严重低头** | 最近 30 秒（10 帧）内 ≥70% 帧 `ema_ratio < 0.3` | 抬头 — 颈椎深度前屈，可能在低头看手机或键盘 |

`ema_ratio` 是头肩高度比经过 EMA 平滑后的值，详见[工作原理](#工作原理)。

### 在座 / 离座状态机

| 情况 | 行为 |
|---|---|
| YOLO 检出人 | 累加在座时间，更新前倾/低头窗口 |
| 连续 ≥ 6 秒（2 帧）未检出 | 判定离座，暂停计时，清空窗口 |
| 离座 < 10 分钟后回来 | 续计久坐时间（短暂喝水不算有效活动） |
| 离座 ≥ 10 分钟后回来 | 久坐时间重置为 0（算有效活动） |

## 配置

所有参数集中在 `main.py` 顶部。

| 参数 | 默认值 | 含义 |
|---|---|---|
| `SAMPLE_INTERVAL` | `3.0` | 采样间隔（秒）|
| `LEAN_RATIO` | `0.5` | 头肩比 < 此值视为轻度前倾 |
| `DEEP_RATIO` | `0.3` | 头肩比 < 此值视为严重低头 |
| `EMA_ALPHA` | `0.4` | EMA 平滑系数（越大跟随越快） |
| `LEAN_WINDOW` | `20` | 前倾判定窗口帧数（× 3s = 60s） |
| `DEEP_WINDOW` | `10` | 低头判定窗口帧数（× 3s = 30s） |
| `HIT_RATIO` | `0.7` | 窗口内命中比例阈值 |
| `SITTING_LIMIT_SEC` | `2400` | 久坐提醒阈值（40 分钟） |
| `ACTIVITY_RESET_SEC` | `600` | 离座 ≥ 此时长重置久坐（10 分钟） |
| `ALERT_COOLDOWN_SEC` | `300` | 同类提醒最短间隔（5 分钟） |

## 调参指南

**头肩比的绝对值跟摄像头位置强相关**，每个人座位环境不同，默认阈值不一定适合你。建议先跑 5 分钟，观察控制台数据：

| 你的端正坐姿数值 | 调整建议 |
|---|---|
| 稳定 0.7+ | 默认阈值 OK |
| 稳定 0.5–0.6 | `LEAN_RATIO=0.4`、`DEEP_RATIO=0.2` |
| 稳定 0.4 以下 | 摄像头放得太低（拍仰角），物理调高摄像头比改阈值更好 |

**抖动**：若 `raw` 与 `平滑` 列剧烈跳动，说明关键点不稳。试试：

- 改善光线（YOLO 在低光下置信度差）
- 摄像头与你保持稳定 70–100 cm 距离
- 调高 `KP_CONF` 到 `0.6` 过滤更严

## 限制与已知问题

诚实说一下做不到的事，避免误用：

1. **分不清"低头玩手机"和"低头看键盘/笔记本"。**
   两者几何特征一致。本项目把它们统一处理为"颈椎前屈过久"——反正对颈椎都是损伤。如果你需要明确识别手机这个物体，需要额外加一个 `yolov8n` 通用检测模型识别 `cell phone` 类。

2. **侧身、转头超过约 60° 时检测失效。**
   双肩任一不可见就直接视为"离座"——会让久坐计时误暂停。如果你工作中经常需要回头跟旁边人交流，这点需要注意。

3. **多人在画面中只取第一个人。**
   不适合公共空间使用。

4. **绝对阈值因人因摄像头而异。**
   `LEAN_RATIO` 默认 0.5 是按摄像头与脸平齐时调的，摄像头放得偏高/偏低需要相应调阈值。

5. **目前只支持 macOS 通知。**
   Windows / Linux 用户需要把 `notify()` 里的 `osascript` 与 `afplay` 替换为对应系统命令（如 `notify-send` / `paplay`），欢迎 PR。
