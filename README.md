# posture-guard

> 本地摄像头颈椎前倾与久坐监测。每 10 秒一帧，纯几何判定，全程离线，不上传任何画面。

久坐和低头是程序员的两大慢性损伤源。市面上的提醒类工具要么是计时器（不知道你是不是真的离座了），要么是依赖云端 AI（隐私堪忧）。这个项目用 [YOLOv11-pose](https://docs.ultralytics.com/tasks/pose/)（导出为 ONNX，~11 MB 本地模型，由 onnxruntime 直推）+ 朴素几何判定，做到三件事：

- 久坐 40 分钟提醒（**真离座**才重置计时，短暂走神不算）
- 颈椎前倾 60 秒提醒
- 严重低头 60 秒提醒

模型完全本地推理，画面不落盘、不联网。在座时每 10 秒采样一帧，**采样时才打开摄像头**；长时间离座自动降频到 10 / 30 分钟一次。平时进程几乎不占 CPU；常驻内存 ~150 MB。

---

## 工作原理

每 10 秒抓一帧 → YOLOv11-pose 先做人物检测，检测到人才输出 17 个 COCO 关键点 → 几个简单几何指标 → 滑动窗口去抖 → 触发提醒。

摄像头**按需开关**：每次抓帧时打开 → 抓一帧 → 立即释放，避免 OpenCV AVFoundation 后台管线持续吃 CPU（这是这类应用最常见的性能陷阱）。代价是摄像头指示灯每 10 秒闪一下（离座 ≥ 5 分钟降到 10 分钟一闪，≥ 30 分钟降到 30 分钟一闪）。

### 如何判定"有人"

YOLOv11-pose 是真正的"先检测人 → 再标关键点"两阶段联合模型——画面里没人时模型直接不输出 detection，从根上不会幻觉关键点。person 置信度 < `PERSON_CONF`（默认 0.5）即判定离座。

**检测到人就无条件算"在座"**，久坐计时正常累加。但头肩比不一定每帧都能算（低头吃东西时鼻/眼/耳的 visibility 会偏低），算不出时让 `head_ratio=None` 跳过这一帧的 EMA 与前倾/低头窗口更新——这样既不会被边缘姿态误判离座，也不会把吃东西误报成"颈部前倾"。

头肩比能算的条件：

1. **双肩**都可见（`KEYPOINT_CONF` ≥ 0.5）
2. 头部 5 点（鼻 + 双眼 + 双耳）至少 **2 个**可见
3. 肩宽 ≥ 30 像素，且头肩比落在 `[-2, 2]` 物理合理区间

> 项目早期用过 MediaPipe BlazePose（lite/heavy），实测在空椅子上仍频繁幻觉出抖动剧烈的"幻觉人"——这是 BlazePose 系架构层面的局限（设计假设是"已知有人，定位关键点"），调阈值挡不住。换 YOLOv11-pose 后这个问题彻底消失。

### 颈椎前倾如何判定

定义 **头肩高度比**：

```
ratio = (肩中点 y - 头中点 y) / 肩宽
```

- 头中点 = 鼻 + 双眼 + 双耳 中所有可见点 y 坐标的平均（最多 5 个点取均值，单点丢失/抖动影响极小）
- 端正坐姿 ≈ **0.7+**（头明显高于肩）
- 轻度低头 ≈ **0.5**
- 严重低头 ≈ **0.3**
- 趴桌 ≈ **负值**

⚠️ 早期版本用 `atan2` 算颈椎与垂直方向的夹角，但 atan2 对水平噪声极为敏感（左右耳交替遮挡、关键点亚像素抖动会被放大成 ±20° 跳动）。改用纯垂直比值后，对水平噪声完全免疫。

### 时序平滑

原始 ratio 仍有少量抖动，再叠一层 EMA：

```
ema = 0.5 * raw + 0.5 * prev_ema
```

判定基于 ema 而不是 raw。

### 滑动窗口防误报

- **轻度前倾**：ema < 0.5，最近 6 帧（60 秒）≥70% 命中才触发
- **严重低头**：ema < 0.3，最近 6 帧（60 秒）≥70% 命中才触发
- 触发"严重低头"时会同时 mute 60 秒内的"轻度前倾"，避免低头看手机时同时收到两条提醒

### 久坐计时

只在"检测到人"时累加：

- 离座 < 10 分钟：回来后**续计**（短暂喝水、找东西不算有效活动）
- 离座 ≥ 10 分钟：回来后**重置为 0**（算作真正起来活动了）
- 任何提醒同类 5 分钟冷却，不会刷屏

### 离座自适应采样

长时间无人时把采样间隔拉长，省 CPU 也省摄像头闪灯次数：

| 离座时长 | 采样间隔 |
|---|---|
| 在座 / 离座 < 5 分钟 | 10 秒 |
| 离座 5 ~ 30 分钟 | 10 分钟 |
| 离座 ≥ 30 分钟 | 30 分钟 |

一旦再次检测到人立即恢复 10 秒。

## 安装

需要 macOS（系统通知依赖 `osascript` + `afplay`）和 Python 3.11+。

```bash
git clone https://github.com/<your-user>/posture-guard.git
cd posture-guard
uv sync   # 或 pip install -r <(uv pip compile pyproject.toml)
```

依赖：`onnxruntime`、`opencv-python`、`numpy`。

## 获取模型

模型文件 `yolo11n-pose.onnx`（~11 MB）需要一次性手动导出。用 `uvx` 临时托管 ultralytics + torch，导出完即可弃用：

```bash
uvx --with onnx --with onnxslim --from ultralytics yolo export model=yolo11n-pose.pt format=onnx imgsz=640
```

> `--with onnx --with onnxslim`：ultralytics 导出 ONNX 时需要这两个包，但它的"自动安装"在 uvx 临时环境里不工作（uvx 创建的 venv 默认不带 pip），所以显式声明。

执行完会在当前目录生成 `yolo11n-pose.onnx`。运行时只依赖 `onnxruntime`，**不需要安装 ultralytics 或 torch**。

## 运行

```bash
uv run main.py
```

首次启动 macOS 会请求摄像头权限；授权后控制台每 10 秒输出一行：

```
[09:12:08] 在座 459s 头肩比 0.65/平滑 0.62 | 推理 6ms
[09:12:18] 在座 469s 头肩比 0.45/平滑 0.55 前倾 | 推理 7ms
[09:12:28] 在座 479s 头肩比 0.18/平滑 0.31 严重低头! | 推理 6ms
[09:35:00] 离座 320s 下次 600s | 推理 7ms
```

`下次 Ns` 标识下一次采样的间隔，方便观察自适应降频是否生效。

`Ctrl+C` 退出。

## 提醒方式

弹出**模态对话框**（必须点"知道了"）+ 本地音效（项目根目录的 `alert.wav`，缺省时跳过播放只弹窗）。10 分钟无操作自动关闭兜底，避免挂死。

如不想要模态弹窗，把 `main.py` 里的 `notify()` 改回 `display notification` 即可（变成系统通知中心横幅）。

## 提醒规则一览

三类提醒，全部同类 5 分钟冷却。

| 类型 | 触发条件 | 文案 |
|---|---|---|
| **久坐** | 连续在座 40 分钟 | 该起来活动了 — 已经连续坐 40 分钟，起来走走吧 |
| **轻度前倾** | 最近 60 秒（6 帧）内 ≥70% 帧 `ema_ratio < 0.5` | 调整坐姿 — 颈部前倾过久，挺直背 |
| **严重低头** | 最近 60 秒（6 帧）内 ≥70% 帧 `ema_ratio < 0.3` | 抬头 — 颈椎深度前屈，可能在低头看手机或键盘 |

触发"严重低头"时会同时 mute 5 分钟内的"轻度前倾"。`ema_ratio` 是头肩高度比经 EMA 平滑后的值，详见[工作原理](#工作原理)。

### 在座 / 离座状态机

| 情况 | 行为 |
|---|---|
| 检出人 | 累加在座时间，更新前倾/低头窗口 |
| 连续 ≥ 20 秒（2 帧）未检出 | 判定离座，暂停计时，清空窗口 |
| 离座 < 10 分钟后回来 | 续计久坐时间（短暂喝水不算有效活动） |
| 离座 ≥ 10 分钟后回来 | 久坐时间重置为 0（算有效活动） |

## 配置

所有参数集中在 `main.py` 顶部。

| 参数 | 默认值 | 含义 |
|---|---|---|
| `SAMPLE_INTERVAL` | `10.0` | 采样间隔（秒）|
| `LEAN_RATIO` | `0.5` | 头肩比 < 此值视为轻度前倾 |
| `DEEP_RATIO` | `0.3` | 头肩比 < 此值视为严重低头 |
| `EMA_ALPHA` | `0.5` | EMA 平滑系数（越大跟随越快） |
| `LEAN_WINDOW` | `6` | 前倾判定窗口帧数（× 10s = 60s） |
| `DEEP_WINDOW` | `6` | 低头判定窗口帧数（× 10s = 60s） |
| `HIT_RATIO` | `0.7` | 窗口内命中比例阈值 |
| `SITTING_LIMIT_SEC` | `2400` | 久坐提醒阈值（40 分钟） |
| `ACTIVITY_RESET_SEC` | `600` | 离座 ≥ 此时长重置久坐（10 分钟） |
| `ALERT_COOLDOWN_SEC` | `300` | 同类提醒最短间隔（5 分钟） |
| `MIN_HEAD_POINTS` | `2` | 头部 5 点至少 N 个可见才算人；真低头被误判为离座时调成 `1` |
| `ABSENT_MEDIUM_SEC` / `INTERVAL_MEDIUM` | `300` / `600.0` | 离座 ≥ 5 分钟，采样间隔切到 10 分钟 |
| `ABSENT_LONG_SEC` / `INTERVAL_LONG` | `1800` / `1800.0` | 离座 ≥ 30 分钟，采样间隔切到 30 分钟 |

## 调参指南

**头肩比的绝对值跟摄像头位置强相关**，每个人座位环境不同，默认阈值不一定适合你。建议先跑 5 分钟，观察控制台数据：

| 你的端正坐姿数值 | 调整建议 |
|---|---|
| 稳定 0.7+ | 默认阈值 OK |
| 稳定 0.5–0.6 | `LEAN_RATIO=0.4`、`DEEP_RATIO=0.2` |
| 稳定 0.4 以下 | 摄像头放得太低（拍仰角），物理调高摄像头比改阈值更好 |

**抖动**：若 `raw` 与 `平滑` 列剧烈跳动，说明关键点不稳。试试：

- 改善光线（低光下检测置信度差）
- 摄像头与你保持稳定 70–100 cm 距离
- 调高 `KEYPOINT_CONF` / `PERSON_CONF` 到 `0.6` 过滤更严

## 限制与已知问题

诚实说一下做不到的事，避免误用：

1. **分不清"低头玩手机"和"低头看键盘/笔记本"。**
   两者几何特征一致。本项目把它们统一处理为"颈椎前屈过久"——反正对颈椎都是损伤。如果你需要明确识别手机这个物体，需要额外加一个目标检测模型识别 `cell phone` 类。

2. **侧身、转头超过约 60° 时检测失效。**
   双肩任一不可见就直接视为"离座"——会让久坐计时误暂停。如果你工作中经常需要回头跟旁边人交流，这点需要注意。

3. **多人在画面中只取第一个人。**
   不适合公共空间使用。

4. **绝对阈值因人因摄像头而异。**
   `LEAN_RATIO` 默认 0.5 是按摄像头与脸平齐时调的，摄像头放得偏高/偏低需要相应调阈值。

5. **目前只支持 macOS 通知。**
   Windows / Linux 用户需要把 `notify()` 里的 `osascript` 与 `afplay` 替换为对应系统命令（如 `notify-send` / `paplay`），欢迎 PR。
